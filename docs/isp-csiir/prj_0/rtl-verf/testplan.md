## 1. 验证概述

### 1.1 验证目标

确保 ISP-CSIIR HLS 模块满足需求规格文档中定义的所有功能、性能和精度要求。

### 1.2 验证范围

| 验证类型 | 覆盖范围 |
|----------|----------|
| 功能验证 | 所有处理阶段的正确性 |
| 边界验证 | 图像边界、极值输入处理 |
| 性能验证 | II=1 目标、吞吐量、延迟 |
| 精度验证 | 位精确模型、定点误差分析 |
| 回归验证 | 代码修改后的持续验证 |

### 1.3 验证方法

| 方法 | 工具 | 目的 |
|------|------|------|
| C 仿真 | Vitis HLS CSIM | 功能正确性验证 |
| 协同仿真 | Vitis HLS CoSim | RTL 功能一致性 |
| 形式验证 | 等价性检查 | C 与 RTL 位精确一致性 |
| 回归测试 | 自动化脚本 | 持续集成验证 |

## 3. 测试向量规格

### 3.1 测试图像规格

| 图像类型 | 尺寸 | 像素范围 | 用途 |
|----------|------|----------|------|
| 平坦图像 | 64x64 | 512 | 基准测试 |
| 水平梯度 | 64x64 | 0-1023 | 梯度测试 |
| 垂直梯度 | 64x64 | 0-1023 | 梯度测试 |
| 棋盘格 | 64x64 | 200, 800 | 边缘测试 |
| 边缘图案 | 64x64 | 100, 512, 900 | 边缘响应 |
| 中心峰值 | 64x64 | 0-1023 | 平滑测试 |
| 随机噪声 | 64x64 | 0-1023 | 压力测试 |
| 小尺寸 | 16x16 | 0-1023 | 边界测试 |

### 3.2 预期输出规格

每种测试图像需要计算预期输出：

1. **Stage 1 预期输出**:
   - `grad`: 梯度值
   - `win_size`: 窗口大小

2. **Stage 2 预期输出**:
   - `avg0_c/u/d/l/r`: 小核方向平均
   - `avg1_c/u/d/l/r`: 大核方向平均

3. **Stage 3 预期输出**:
   - `blend0_grad`: 小核融合
   - `blend1_grad`: 大核融合

4. **Stage 4 预期输出**:
   - `dout`: 最终输出像素

## 5. 验证流程

### 5.0 中间阶段验证流程（新增）

为确保 HLS 模型正确性，我们实现了 Python 定点化模型与 HLS 模型的中间阶段数据对比验证。

```
┌─────────────────┐     ┌─────────────────┐
│  测试向量生成    │────→│  Python 模型    │
│  generate_      │     │  compute_       │
│  test_vectors.py│     │  expected.py    │
└─────────────────┘     └────────┬────────┘
                                 │
                                 ▼
                        ┌────────────────┐
                        │ 中间数据输出    │
                        │ export_        │
                        │ intermediate.py│
                        └────────┬───────┘
                                 │
┌─────────────────┐              │
│  HLS 测试平台   │              │
│  tb_isp_csiir_  │              │
│  intermediate   │              │
└────────┬────────┘              │
         │                       │
         ▼                       ▼
┌────────────────┐      ┌────────────────┐
│  HLS 中间数据   │      │ Python 中间数据 │
│  intermediate/ │      │ intermediate/   │
│  hls/          │      │ python/         │
└───────┬────────┘      └────────┬────────┘
        │                        │
        └────────────┬───────────┘
                     │
                     ▼
           ┌──────────────────┐
           │  数据比对脚本    │
           │ compare_         │
           │ intermediate.py  │
           └────────┬─────────┘
                    │
                    ▼
           ┌──────────────────┐
           │   验证报告       │
           └──────────────────┘
```

**中间数据字段定义：**

| 阶段 | 字段 | 类型 | 容差 |
|------|------|------|------|
| Stage 1 | grad | U12 | ±1 |
| Stage 1 | win_size | U6 | 0 (精确) |
| Stage 1 | grad_h/v | S14 | ±1 |
| Stage 2 | avg0_*, avg1_* | S11 | ±2 |
| Stage 3 | blend0/1_grad | S11 | ±3 |
| Output | output_pixel | U10 | ±2 |

**验证执行脚本：**

```bash
# 1. 生成测试向量
cd verification/scripts
python generate_test_vectors.py

# 2. 导出 Python 中间数据
python export_intermediate.py

# 3. 运行 HLS 测试平台（需要 HLS 工具）
cd ../../hls
# 使用 Vitis HLS:
vitis_hls -f run_tb_intermediate.tcl
# 或使用 g++ 编译测试：
g++ -std=c++11 -I$HLS_HOME/include \
    -o tb_intermediate tb_isp_csiir_intermediate.cpp \
    isp_csiir.cpp stage*.cpp div_utils.cpp
./tb_intermediate

# 4. 比对中间数据
cd ../verification/scripts
python compare_intermediate.py
```

### 5.1 单元验证流程

```
1. 生成测试向量
   $ python generate_test_vectors.py

2. 运行 C 仿真
   $ vitis_hls -f run_csim.tcl

3. 分析仿真结果
   $ python analyze_results.py

4. 检查覆盖率
   - 功能覆盖率
   - 代码覆盖率
```

### 5.2 回归验证流程

```
1. 代码修改后触发
2. 运行所有测试用例
3. 对比预期输出
4. 生成回归报告
5. 通知失败用例
```

### 5.3 HLS 综合验证流程

```
1. 运行 HLS 综合
   $ vitis_hls -f run_synthesis.tcl

2. 检查综合报告
   - II 目标达成
   - 资源使用
   - 时序估计

3. 运行协同仿真
   $ vitis_hls -f run_cosim.tcl

4. 导出 RTL
   $ vitis_hls -f export_rtl.tcl
```

## 8. 风险与缓解措施

### 8.1 验证风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 测试向量不完整 | 漏测功能 | 中 | 覆盖率分析 |
| 预期输出计算错误 | 误判结果 | 低 | 交叉验证 |
| HLS 工具版本差异 | 结果不一致 | 低 | 版本锁定 |
| 除法精度问题 | 输出误差 | 中 | 精度测试 |

### 8.2 缓解计划

1. 定期审查测试覆盖率
2. 多组独立计算预期输出
3. 锁定工具版本
4. 增加精度边界测试用例
